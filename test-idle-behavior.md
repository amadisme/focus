# 测试智能闲置检测

## 修复后的逻辑

### 三种状态判定

1. **活跃工作**（闲置时间 < 5秒）
   - ✅ 计时器累加
   - 示例：正在打字、移动鼠标、操作电脑

2. **暂停状态**（5秒 ≤ 闲置时间 < 5分钟）
   - ⏸️ 计时器暂停（不累加，也不重置）
   - 示例：阅读文档、思考、看屏幕但不操作

3. **离座状态**（闲置时间 ≥ 5分钟）
   - 🔄 计时器重置为0
   - 示例：去喝水、开会、离开电脑

## 快速测试步骤

### 1. 启动应用
```bash
npm start
```

### 2. 测试场景A：持续工作
1. 持续移动鼠标或打字（保持闲置时间 < 5秒）
2. 观察时间是否持续累加
3. **预期**：时间每秒增加1秒

### 3. 测试场景B：停止操作10秒
1. 工作一会儿后，完全停止操作（不动鼠标、不打字）
2. 等待10秒
3. 观察计时器是否暂停
4. **预期**：时间停止累加，保持不变

### 4. 测试场景C：恢复操作
1. 在暂停后，移动鼠标
2. 观察计时器是否恢复
3. **预期**：时间继续从之前的值累加

### 5. 测试场景D：离座5分钟（快速测试）
**临时修改配置**：
```javascript
// 在 src/main.js 中修改
const RESET_THRESHOLD = 30;  // 改为30秒测试
const ACTIVE_THRESHOLD = 3;  // 改为3秒测试
```

重启应用后：
1. 工作一会儿（移动鼠标）
2. 完全停止操作30秒
3. 观察计时器是否重置为0
4. **预期**：30秒后，控制台显示"检测到用户离座..."，时间归零

## 关键区别

### ❌ 修复前（单纯计时工具）
```
时间0秒 → 启动应用
时间1秒 → 无论是否操作，+1秒
时间2秒 → 无论是否操作，+1秒
...
只有离座5分钟才重置
```

### ✅ 修复后（真正的智能工具）
```
时间0秒 → 启动应用
时间1秒 → 正在打字，+1秒
时间2秒 → 正在打字，+1秒
时间3秒 → 停止操作（阅读），暂停
时间3秒 → 还在阅读，保持3秒
时间4秒 → 开始打字，+1秒
...
离座5分钟 → 重置为0
```

## 配置参数说明

```javascript
// src/main.js
const TARGET_WORK_TIME = 45 * 60;    // 45分钟
const RESET_THRESHOLD = 5 * 60;      // 5分钟：离座重置
const ACTIVE_THRESHOLD = 5;          // 5秒：判定为活跃
const REMIND_PHASE_2_TIME = 50 * 60; // 50分钟：强提醒
```

### 为什么选择5秒？
- **太短（如1秒）**：阅读时眼睛停顿1秒就暂停，体验不好
- **太长（如30秒）**：用户停止操作30秒还在累加，不够智能
- **5秒刚好**：既能识别真正的工作，又能容忍短暂停顿
