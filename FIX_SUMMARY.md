# 🔧 重大修复：智能计时逻辑

## 问题发现

用户发现了一个**关键问题**：原实现虽然集成了 `desktop-idle` 系统闲置时间检测，但**没有真正利用**它来实现智能计时。

### ❌ 修复前的逻辑（错误）

```javascript
tick() {
  const idleTime = this.getIdleTime();
  
  if (idleTime >= this.resetThreshold) {  // >= 5分钟
    // 重置计时器
    this.reset();
  } else {  // < 5分钟
    // ❌ 无论闲置多久，只要 < 5分钟就一直累加
    this.accumulatedWorkTime += 1;
  }
}
```

**问题**：
1. 用户停止操作10秒、30秒、1分钟、4分钟，计时器**还在继续跑**
2. 这就变成了一个**单纯的计时工具**，跟普通定时器没区别
3. 没有体现"智能"，违背了原始需求

### 实际场景举例

#### 场景1：阅读文档
```
用户行为: 工作10分钟 → 阅读文档2分钟（不动鼠标）→ 继续工作
旧逻辑: 10分钟 → 继续累加到12分钟 → 继续累加
❌ 问题: 阅读时没有操作，但计时器还在跑
```

#### 场景2：思考问题
```
用户行为: 写代码20分钟 → 思考1分钟 → 继续写代码
旧逻辑: 20分钟 → 继续累加到21分钟 → 继续累加
❌ 问题: 思考时没有操作，但计时器还在跑
```

**结论**：旧逻辑只是一个"定时启动的秒表"，不是真正的智能工具。

## ✅ 修复后的逻辑（正确）

### 新的三态判定系统

```javascript
tick() {
  const idleTime = this.getIdleTime();
  
  // 情况1: 闲置时间 >= 5分钟 → 完全重置
  if (idleTime >= this.resetThreshold) {
    this.reset();
    this.state = 'idle';
  }
  // 情况2: 闲置时间 < 5秒 → 用户活跃，累加工作时间
  else if (idleTime < this.activeThreshold) {
    this.accumulatedWorkTime += 1;
    this.state = 'working';
  }
  // 情况3: 5秒 <= 闲置时间 < 5分钟 → 暂停累加
  else {
    // 不累加，但也不重置
    // 保持当前累计时间
  }
}
```

### 三种状态详解

| 闲置时间 | 状态 | 动作 | 场景示例 |
|---------|------|------|---------|
| < 5秒 | 活跃工作 | ✅ 计时累加 | 打字、移动鼠标 |
| 5秒 ~ 5分钟 | 暂停状态 | ⏸️ 计时暂停 | 阅读、思考 |
| ≥ 5分钟 | 离座状态 | 🔄 计时重置 | 开会、喝水 |

### 新参数：ACTIVE_THRESHOLD

```javascript
const ACTIVE_THRESHOLD = 5;  // 5秒：判定为活跃使用的阈值
```

**为什么是5秒？**
- **< 1秒**：太敏感，眼睛停顿就暂停
- **5秒**：刚好，既能识别真正工作，又容忍短暂停顿
- **> 30秒**：太迟钝，用户停止操作很久还在累加
