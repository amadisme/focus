# 🎯 优化：将重置阈值从5分钟改为10分钟

## 用户需求

> "增加一个逻辑，如果闲置时间持续10分钟以上，重置已累计的专注时间，因为这时候可能已经离开工位了，已经达到了休息的目的，下次回来重新计时即可。"

## 优化内容

### 修改前（5分钟重置）
```javascript
const RESET_THRESHOLD = 5 * 60;  // 5分钟
```

**问题**：
- 喝水5分钟回来 → 计时器归零 ❌
- 上厕所5分钟回来 → 计时器归零 ❌
- 接个电话5分钟 → 计时器归零 ❌
- **短暂休息也会重置，丢失工作成果**

### 修改后（10分钟重置）
```javascript
const RESET_THRESHOLD = 10 * 60;  // 10分钟
```

**优势**：
- 喝水5分钟回来 → 计时暂停，保留工作时间 ✅
- 上厕所5分钟回来 → 计时暂停，保留工作时间 ✅
- 接电话8分钟 → 计时暂停，保留工作时间 ✅
- 开会15分钟 → 计时重置，已达到休息目的 ✅
- **既保留短暂休息的工作成果，又识别真正的长时间离开**

## 完整的三态逻辑

| 闲置时间 | 状态 | 动作 | 场景示例 |
|---------|------|------|---------|
| **< 5秒** | 🟢 活跃工作 | ✅ 计时累加 | 打字、移动鼠标 |
| **5秒 ~ 10分钟** | 🟡 暂停状态 | ⏸️ 计时暂停 | 阅读、思考、喝水、上厕所 |
| **≥ 10分钟** | 🔴 离座状态 | 🔄 计时重置 | 开会、午休、外出 |

## 实际场景分析

### 场景1：工作中的短暂休息（新逻辑优势明显）

```
时间轴：
0:00 - 30:00  → 专注工作30分钟
30:00 - 35:00 → 离开喝水5分钟
35:00 - 45:00 → 继续工作10分钟

5分钟阈值（旧）:
30分钟 → 重置为0 → 从0开始累加10分钟 = 10分钟
❌ 丢失了前面30分钟的工作成果

10分钟阈值（新）:
30分钟 → 暂停 → 继续累加10分钟 = 40分钟
✅ 保留了前面30分钟的工作成果
```

### 场景2：真正的长时间休息（两种阈值结果相同）

```
时间轴：
0:00 - 40:00  → 专注工作40分钟
40:00 - 55:00 → 离开开会15分钟
55:00 - 60:00 → 继续工作5分钟

5分钟阈值:
40分钟 → 重置为0 → 从0开始累加5分钟 = 5分钟
✅ 开会15分钟，已达到休息目的

10分钟阈值:
40分钟 → 重置为0 → 从0开始累加5分钟 = 5分钟
✅ 开会15分钟，已达到休息目的
```

## 设计理念

### 保护用户的工作成果
- 短暂休息（< 10分钟）不应该清零累计时间
- 用户需要的是"连续工作45分钟后提醒"
- 中间喝水、上厕所应该算作工作的一部分

### 识别真正的休息
- 超过10分钟的离开通常是有意的休息
- 开会、午休、外出都超过10分钟
- 这时候重置是合理的，因为已经达到休息目的

### 符合实际工作场景
```
真实的工作流程：
8:00 - 9:30   工作 (90分钟)
9:30 - 9:35   喝水 (5分钟)    → 应该暂停，不重置
9:35 - 11:00  工作 (85分钟)
11:00 - 12:00 开会 (60分钟)   → 应该重置，已休息
12:00 - 13:00 午休 (60分钟)   → 应该重置，已休息
13:00 - 14:30 工作 (90分钟)
14:30 - 14:35 上厕所 (5分钟)  → 应该暂停，不重置
14:35 - 16:00 工作 (85分钟)
```

## 可调整性

如果用户觉得10分钟仍然不够合理，可以轻松调整：

```javascript
// src/main.js

// 更短：适合频繁短会的工作环境
const RESET_THRESHOLD = 8 * 60;  // 8分钟

// 更长：适合需要长时间专注的工作
const RESET_THRESHOLD = 15 * 60;  // 15分钟

// 推荐：适合大多数办公场景
const RESET_THRESHOLD = 10 * 60;  // 10分钟 ✅
```
