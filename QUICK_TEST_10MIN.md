# 快速测试 10 分钟重置逻辑

## 方法1：使用临时快速测试配置

为了快速测试 10 分钟重置逻辑，可以临时修改阈值：

### 1. 临时修改配置
编辑 `src/main.js`：

```javascript
// 临时测试配置（测试后记得改回来！）
const TARGET_WORK_TIME = 20;         // 20秒（原45分钟）
const RESET_THRESHOLD = 30;          // 30秒（原10分钟）
const REMIND_PHASE_2_TIME = 25;      // 25秒（原50分钟）
const ACTIVE_THRESHOLD = 3;          // 3秒（原5秒）
```

### 2. 启动应用
```bash
npm start
```

### 3. 测试场景

#### 场景A：活跃工作（< 3秒闲置）
1. 持续移动鼠标或打字
2. 观察时间每秒累加
3. **预期**：时间持续增长 ✅

#### 场景B：暂停状态（3秒 ~ 30秒闲置）
1. 工作10秒后，完全停止操作
2. 等待10秒（保持闲置）
3. 观察时间是否暂停
4. **预期**：时间停在10秒不变 ✅
5. **控制台输出**：`用户暂时停止操作（闲置 XX 秒），计时暂停`

#### 场景C：恢复工作
1. 在暂停后，移动鼠标
2. 观察时间是否继续累加
3. **预期**：从10秒继续增长到11秒、12秒... ✅

#### 场景D：离座重置（≥ 30秒闲置）
1. 工作15秒后，完全停止操作
2. 等待30秒以上
3. 观察时间是否重置为0
4. **预期**：时间归零 ✅
5. **控制台输出**：`检测到用户离座/开会超过30秒，专注计时器清零重置。`

### 4. 完整测试流程（推荐）

```
时间轴    | 操作              | 预期结果
---------|-------------------|------------------
0-10秒   | 持续打字          | 0:00 → 0:10 ✅
10-20秒  | 停止操作10秒      | 0:10 → 0:10 (暂停) ✅
20-25秒  | 继续打字          | 0:10 → 0:15 ✅
25-60秒  | 停止操作35秒      | 0:15 → 0:00 (重置) ✅
60-65秒  | 重新打字          | 0:00 → 0:05 ✅
```

### 5. 测试完成后
**重要**：恢复正常配置！

```javascript
// 恢复正常配置
const TARGET_WORK_TIME = 45 * 60;    // 45分钟
const RESET_THRESHOLD = 10 * 60;     // 10分钟
const REMIND_PHASE_2_TIME = 50 * 60; // 50分钟
const ACTIVE_THRESHOLD = 5;          // 5秒
```

## 故障排除

### 问题：时间一直在累加，没有暂停
**原因**：可能鼠标有微小移动（触摸板、鼠标灵敏度）
**解决**：
1. 完全不要触碰键盘鼠标
2. 或者临时提高 `ACTIVE_THRESHOLD` 到 10 秒

### 问题：没有看到控制台日志
**原因**：后台运行时看不到
**解决**：
1. 从终端启动 `npm start`
2. 或使用开发模式 `npm run dev`

### 问题：测试太慢（等待10分钟）
**解决**：使用方法1的临时配置，将阈值降低到 30 秒
